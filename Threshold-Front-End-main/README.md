# Threshold Frontend

This project, named "threshold-frontend," is a React-based web application. It leverages a robust set of tools and libraries to ensure code quality, ease of development, and efficient performance.

## Key Dependencies

- **React (v18.2.0)**: The core library for building the user interface.
- **Redux (v4.2.1) & Redux Toolkit (v1.9.7)**: For state management within the app.
- **TypeScript (v4.9.5)**: Used for adding type safety to the JavaScript code.
- **Webpack (v5.89.0)**: For bundling the application modules.

## Development Tools

- **ESLint (v8.54.0)**: For enforcing coding standards and identifying potential errors.
- **Prettier (v3.1.0)**: An opinionated code formatter to maintain consistent code styling.
- **Husky (v8.0.0)**: To apply Git hooks for actions like pre-commit linting.
- **npm-run-all (v4.1.5)**: For running multiple npm scripts in parallel or sequentially.

## Available Scripts

In the project directory, you can run:

### `npm start`

Runs the app in development mode. Open [http://localhost:3000](http://localhost:3000) to view it in your browser. The page will reload with edits, and lint errors will show in the console.

### `npm run build`

Builds the app for production to the `build` folder.

### `npm test`

Launches the test runner in the interactive watch mode.

### `npm run format`

Runs Prettier to format the code in the `src` directory.

### `npm run lint`

Runs ESLint for TypeScript files in the project.

### `npm run validate`

Runs multiple checks including linting, type checking, and format validation in parallel.

### `npm run prepare`

Sets up Husky for Git hooks.
